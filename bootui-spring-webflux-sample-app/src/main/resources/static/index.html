Spring Boot 4 WebFlux demo

Welcome to the BootUI WebFlux sample app

This app demonstrates BootUI running on the reactive stack (Netty, WebFlux): cached lookups, blocking JDBC pushed off the event loop, Flyway/Liquibase-managed data, a scheduled task, and enough runtime state for BootUI to inspect.

Open BootUI

Sample action lab

Run a few safe requests to generate data for BootUI panels. Then open the linked panels to see how BootUI explains the same reactive app from different angles.

Calls GET /api/notes to run a blocking JDBC query off the event loop.

Inspect: SQL Trace, Database Connection Pools, HTTP Exchanges

Calls GET /api/notes/1 to fetch a single row by id.

Inspect: SQL Trace, HTTP Exchanges

Calls GET /api/greetings/chained so BootUI can show configuration-backed, cached behavior.

Inspect: Configuration, Cache, HTTP Probe

Calls GET /api/sample/boom, which throws on purpose to record a server-side failure.

Inspect: Exceptions, Log Tail, Traces

Calls GET /api/sample/metrics-burst to record a custom counter and timer several times.

Inspect: Metrics

Calls GET /api/sample/allocate to briefly allocate a bounded heap buffer.

Inspect: Live Memory, Heap Dump, JVM Tuning

Calls GET /api/sample/slow, which sleeps briefly on a worker thread.

Inspect: Threads, Metrics, HTTP Exchanges

Calls GET /api/sample/pool-stress to run several concurrent read-only queries.

Inspect: Database Connection Pools, SQL Trace

Calls GET /api/sample/chained, which nests spans across a notes read and a greeting call.

Inspect: Traces, HTTP Exchanges, SQL Trace
